## Why role-based guides

Different teams use Conviso Platform for different outcomes. These guides help each profile focus on the right workflows, metrics, and decisions.

## Available profiles

- [DevSecOps](./role-based-guide-devsecops)
- [Developer](./role-based-guide-developer)
- [Penetration Tester](./role-based-guide-penetration-tester)
- [AppSec Engineer](./role-based-guide-appsec-engineer)
- [AppSec Manager](./role-based-guide-appsec-manager)
- [Security Manager](./role-based-guide-security-manager)

## How to use this section

1. Start with your primary role.
2. Follow the weekly and monthly routines.
3. Track the suggested KPIs.
4. Use linked docs to go deeper into each operation.

## Shared baseline for all roles

- Keep integrations healthy (SCM, CI/CD, notifications, issue trackers).
- Keep findings triage active and SLA visible.
- Use standardized screenshots and writing guidelines for internal documentation updates.
- Review release notes frequently for behavior changes.
